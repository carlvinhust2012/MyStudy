# QLC（Quad-Level Cell，四层单元）存储介质技术分析

## 1. 一句话概括

QLC 是一种 NAND 闪存技术，每个存储单元存放 4 bit 数据，通过牺牲耐久性和性能换取更大的存储容量和更低的单位成本。

## 2. NAND 闪存演进：从 SLC 到 QLC

NAND 闪存按每个存储单元（Cell）存放的 bit 数分类:

| 类型 | 全称 | 每 Cell bit 数 | 电压状态数 | 相对容量 | 相对耐久性 | 相对价格 |
|---|---|---|---|---|---|---|
| **SLC** | Single-Level Cell | 1 bit | 2 个 | 1x | 最高 | 最贵 |
| **MLC** | Multi-Level Cell | 2 bit | 4 个 | 2x | 较高 | 较贵 |
| **TLC** | Triple-Level Cell | 3 bit | 8 个 | 3x | 中等 | 中等 |
| **QLC** | Quad-Level Cell | 4 bit | 16 个 | 4x | 最低 | 最便宜 |
| **PLC** | Penta-Level Cell | 5 bit | 32 个 | 5x | 极低 | 更便宜（研发中） |

```
容量 ↑  SLC  →  MLC  →  TLC  →  QLC  →  PLC
耐久性 ↓  ↓        ↓       ↓       ↓       ↓
价格   ↓  ↓        ↓       ↓       ↓       ↓

  1bit   2bit    3bit    4bit    5bit
  ───    ────    ────    ────    ────
  2态    4态     8态     16态    32态
```

## 3. QLC 存储原理

### 3.1 电压状态编码

每个存储单元通过不同的电荷量（电压等级）来表示数据:

```
SLC（1 bit）:  2 个电压区间
  ┌─────┐ ┌─────┐
  │  0  │ │  1  │
  └─────┘ └─────┘

QLC（4 bit）:  16 个电压区间
  ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐
  │00││01││02││03││04││05││06││07││08││09││10││11││12││13││14││15│
  └──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘└──┘
  4bit = 0000 到 1111，共 16 种状态

问题: 16 个区间挤在同一个电压范围内
  → 每个区间的电压差值非常小
  → 更容易受干扰导致读错
  → 需要更复杂的纠错（ECC）和读电压校准
```

### 3.2 为什么能省钱

```
同一块硅芯片面积下:

  SLC: 每个 Cell 存 1 bit → 总容量 = Cell 数量 x 1
  QLC: 每个 Cell 存 4 bit → 总容量 = Cell 数量 x 4

  同样的物理芯片，QLC 容量是 SLC 的 4 倍
  → 单位 GB 成本大幅降低
```

## 5. QLC 的核心优点

### 5.1 单位成本最低

QLC 是目前单位 GB 成本最低的固态存储，已接近 HDD 的水平:

```
2024 年大致价格参考（每 GB）:
  SLC SSD:    $5-10
  MLC SSD:    $2-5
  TLC SSD:    $0.5-1.5
  QLC SSD:    $0.1-0.3     ← 接近 HDD
  HDD:        $0.02-0.05
```

### 5.2 大容量

单盘可达 8TB-128TB（甚至 192TB），适合存储海量冷数据和温数据。

### 5.3 功耗低

相比同容量的 HDD 服务器，QLC SSD 功耗大幅降低:

```
100TB 存储:
  HDD 服务器（约 50 块 2TB HDD）:   ~600W
  QLC SSD 服务器（约 4 块 30TB）:   ~100W
  节省约 83% 功耗和散热成本
```

### 5.4 随机访问能力

虽然 QLC 随机写不如 TLC，但随机读仍然远超 HDD（50000 IOPS vs 200 IOPS）。

## 6. QLC 的核心缺点

### 6.1 耐久性极低

这是 QLC 最大的短板，DWPD（Disk Writes Per Day）仅 0.1-1:

```
耐久性对比（DWPD）:

  SLC:  ████████████████████  10-30
  MLC:  ██████████            3-10
  TLC:  █████                  1-3
  QLC:  █                      0.1-1    ← 容易写坏

意味着:
  一块 16TB QLC SSD（DWPD=0.3，5年质保）:
  总写入量 = 16TB x 0.3 x 365 x 5 = 8,760 TB
  每天只能写 16TB x 0.3 = 4.8TB

  对比 TLC（DWPD=1）:
  每天可写 16TB x 1 = 16TB（是 QLC 的 3.3 倍）
```

### 6.2 写性能严重退化

QLC 有一个 TLC 所没有的严重问题: **写放大和性能悬崖**。

```
QLC 写入的三种模式:

  1. SLC Cache 命中（正常写入）:
     写入数据先进入 SLC Cache → 速度与 TLC 相当
     带宽: 1-3 GB/s
     前提: Cache 有空间

  2. SLC Cache 用尽 → 直接写 QLC:
     需要执行较慢的 QLC 编程操作
     带宽: 暴降到 100-200 MB/s（下降 90%+）

  3. 需要垃圾回收（GC）时:
     同时进行 GC + 写入
     带宽: 可能进一步降到 50-100 MB/s

     ┌─────────────────────────────────────────────┐
     │          QLC 写性能悬崖示意图                  │
     ├─────────────────────────────────────────────┤
     │                                             │
     │ 带宽                                       │
     │  3GB/s |████████                            │
     │        |        SLC Cache 区间               │
     │  1GB/s |        ────────────                 │
     │        |                       |             │
     │  200M  |                       |████         │
     │        |                       |直接写QLC     │
     │  100M  |                       |    ████████ │
     │        |                       |    GC+写入   │
     │        +───────────────────────+────────────→ │
     │        0     Cache用尽    写入量              │
     │                                             │
     └─────────────────────────────────────────────┘
```

### 6.3 读性能波动（Read Disturb）

QLC 存储单元之间电压差很小，长时间不读数据可能出现"电荷泄漏"，需要周期性地读校准:

```
正常状态:  读一次就拿到数据（TLC 同样如此）
异常状态:  首次读取返回错误 → 需要重试并校准读电压 → 延迟从 100us 飙升到 10ms

影响:
  - 冷数据（很久没读过的）首次访问延迟极高
  - 对延迟敏感的业务（数据库、KV 存储）不友好
```

### 6.4 纠错开销大

QLC 需要 LDPC（Low-Density Parity-Check）更高级别的纠错:

```
SLC: 简单 ECC 即可（1 bit 纠错）
TLC: 中等强度 LDPC
QLC: 最高强度 LDPC + 更多校验位 + 更复杂的解码计算
     → 读的时候需要更多计算，增加延迟
     → 纠错码占用更多空间（Overhead 更大）
```

### 6.5 不适合随机写密集场景

随机写会导致大量垃圾回收，加速 QLC 磨损:

```
高随机写场景（数据库 WAL、日志系统）:
  → 触发频繁 GC → 擦写放大 → 寿命快速消耗
  → 1 块 16TB QLC 盘可能在 1-2 年内达到耐久性上限

推荐: 随机写密集场景用 TLC 或 SCM
```

## 7. QLC 的适用场景与不适用场景

### 7.1 适合的场景

| 场景 | 原因 |
|---|---|
| 冷数据存储 | 写入后很少修改，对耐久性要求低 |
| 对象存储后端 | 顺序写为主，读多写少 |
| 大数据分析 | 顺序扫描，随机 IO 少 |
| 媒体存储（视频、图片）| 大文件顺序读写，写一次读多次 |
| CDN 节点 | 缓存内容写少读多 |
| 备份存储 | 写入后几乎只读 |
| VDI（虚拟桌面）| 系统盘写一次，后续读为主 |

### 7.2 不适合的场景

| 场景 | 原因 |
|---|---|
| 数据库（MySQL、Redis）| 随机写密集，耐久性不够 |
| WAL 日志 | 频繁追加写，磨损快 |
| 虚拟化平台系统盘 | 随机 IO 混合，性能不稳定 |
| 高频交易 | 延迟波动大，不可接受 |
| 日志收集系统（Kafka 等）| 频繁追加写+清理，GC 压力大 |

## 10. 未来趋势: PLC（五层单元）

下一代 NAND 技术 PLC（Penta-Level Cell）每个 Cell 存 5 bit（32 个电压状态）:

```
趋势: 更大的容量，更低的成本，但也更差的耐久性和性能

        SLC → MLC → TLC → QLC → PLC（研发中）
        1bit  2bit  3bit  4bit  5bit
        2态   4态   8态   16态  32态

挑战:
  - 32 个电压区间的识别极其困难
  - 耐久性可能降至 0.01-0.1 DWPD
  - 需要 AI 辅助的电压校准和纠错
  - 预计 2026-2028 年量产
```

## 11. 总结

**QLC = 大容量 + 低成本 + 高顺序读性能**，代价是 **耐久性低 + 写性能不稳定（SLC Cache 悬崖） + 读延迟波动**。QLC 适合"写一次、读多次"的温冷数据存储场景（对象存储、备份、大数据），不适合随机写密集的在线业务（数据库、Kafka）。在实际架构中，QLC 通常作为分层存储的底层，与 TLC/SCM 组合使用。
