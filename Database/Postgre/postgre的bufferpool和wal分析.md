# PostgreSQL BufferPool 与 WAL 机制分析

## 概述

本文档总结 PostgreSQL 的两个核心问题：
1. PostgreSQL 的 Buffer Pool 机制
2. WAL (Write-Ahead Log) 中保存的内容

---

## 问题 1：PostgreSQL 的 Buffer Pool 机制

### 答案

**是的**，PostgreSQL 也是先把数据写到内存池，再异步刷盘。

### 写入流程

```
用户写入请求
    │
    ▼
1. 写入 WAL Buffer (内存)
   - 保证持久性
   - 延迟: ~μs 级
    │
    ▼
2. 写入 Shared Buffers (内存)
   - 修改数据页
   - 标记为 dirty
   - 延迟: ~μs 级
    │
    │ 返回成功给用户
    ▼
3. 后台刷盘
   ├── wal_writer: WAL buffer → WAL 文件
   ├── bg_writer: dirty pages → 数据文件
   └── checkpointer: checkpoint 时刷盘
```

### 关键参数

```sql
-- 内存缓冲大小
shared_buffers = 128MB          -- 数据页缓存
wal_buffers = 16MB              -- WAL 日志缓存

-- 刷盘策略
wal_writer_delay = 200ms        -- WAL 刷盘间隔
bgwriter_delay = 200ms          -- 后台写进程间隔
checkpoint_timeout = 5min       -- checkpoint 间隔
```

### 写入顺序

```
PostgreSQL 写入顺序 (WAL 先写原则):

1. 写 WAL 到 WAL Buffer
2. 写数据到 Shared Buffers (标记 dirty)
3. 返回成功
4. WAL Buffer 刷到磁盘 (持久化保证)
5. Shared Buffers 刷到磁盘 (可延迟)

核心原则: WAL 先于数据页刷盘
```

## 问题 2：WAL 中保存的是什么内容？

### 答案

WAL 记录的是**数据变更的操作日志**，不是完整数据。

### WAL 记录类型

```
1. 数据变更 (最常见)
   ├── INSERT: 插入的行数据
   ├── UPDATE: 旧值 + 新值 (或只记录新值)
   └── DELETE: 被删除行的标识

2. 事务操作
   ├── XLOG_XACT_COMMIT: 事务提交
   ├── XLOG_XACT_ABORT: 事务回滚
   └── XLOG_XACT_PREPARE: 2PC 准备阶段

3. DDL 操作
   ├── CREATE TABLE: 表结构定义
   ├── DROP TABLE: 删除操作
   └── ALTER TABLE: 修改操作

4. Checkpoint 记录
   └── XLOG_CHECKPOINT: checkpoint 位置信息
```

### WAL 记录结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        WAL 记录结构 (XLogRecord)                         │
├─────────────────────────────────────────────────────────────────────────┤
│  Header (固定 24B)                                                       │
│  ├── xl_tot_len: 记录总长度                                              │
│  ├── xl_xid: 事务 ID                                                     │
│  ├── xl_prev: 前一条记录位置                                             │
│  ├── xl_info: 记录类型信息                                               │
│  ├── xl_rmid: 资源管理器 ID                                              │
│  ├── xl_crc: CRC 校验和                                                  │
│  └── xl_prev: 前一条记录的 LSN                                           │
│  Data (变长)                                                             │
│  └── 具体的变更数据                                                       │
└─────────────────────────────────────────────────────────────────────────┘
```

### INSERT 操作的 WAL 内容

```
INSERT INTO users (id, name) VALUES (1, 'Alice');

WAL 记录内容:
┌─────────────────────────────────────────────────────────────────────────┐
│  Header:                                                                │
│    xl_info = XLOG_HEAP_INSERT                                          │
│    xl_xid = 12345 (事务 ID)                                             │
│                                                                          │
│  Data:                                                                  │
│    ├── 表 OID: 16384                                                   │
│    ├── 页号: 0                                                          │
│    ├── 元组偏移: 8                                                      │
│    └── 元组数据: (1, 'Alice')                                           │
│             ├── id: 1 (4B)                                             │
│             └── name: 'Alice' (6B)                                     │
└─────────────────────────────────────────────────────────────────────────┘
```

### UPDATE 操作的 WAL 内容

```
UPDATE users SET name = 'Bob' WHERE id = 1;

WAL 记录内容:
┌─────────────────────────────────────────────────────────────────────────┐
│  Header:                                                                │
│    xl_info = XLOG_HEAP_UPDATE                                          │
│    xl_xid = 12346                                                       │
│                                                                          │
│  Data:                                                                  │
│    ├── 表 OID: 16384                                                   │
│    ├── 旧元组位置: (页号, 偏移)                                         │
│    ├── 旧元组数据: (1, 'Alice')    ← 用于回滚                           │
│    └── 新元组数据: (1, 'Bob')      ← 用于重放                           │
└─────────────────────────────────────────────────────────────────────────┘
```

### WAL vs 数据文件

```
数据文件 (8KB 页):
┌────────────────────────────────────────────────────────────────────────┐
│ Page Header (24B)                                                      │
│ - pd_lsn: 最后修改该页的 WAL 位置                                       │
│ - pd_lower: 空闲空间起始                                                │
│ - pd_upper: 空闲空间结束                                                │
├────────────────────────────────────────────────────────────────────────┤
│ 行指针数组                                                              │
├────────────────────────────────────────────────────────────────────────┤
│ 行数据: id=1, name='Alice'                                              │
│ 行数据: id=2, name='Bob'                                                │
└────────────────────────────────────────────────────────────────────────┘

WAL 文件 (16MB):
┌────────────────────────────────────────────────────────────────────────┐
│ XLogRecord 1: INSERT (id=1, 'Alice')                                   │
│ XLogRecord 2: INSERT (id=2, 'Bob')                                     │
│ XLogRecord 3: COMMIT (xid=12345)                                       │
│ XLogRecord 4: UPDATE (id=1, 'Bob')                                     │
│ XLogRecord 5: CHECKPOINT                                               │
│ ...                                                                     │
└────────────────────────────────────────────────────────────────────────┘
```

### WAL 的作用

| 作用 | 说明 |
|------|------|
| **崩溃恢复** | 重放 WAL 恢复数据到一致状态 |
| **流复制** | 发送 WAL 到 standby 节点 |
| **时间点恢复** | PITR，恢复到任意时间点 |
| **逻辑解码** | 解析 WAL 做数据同步 |

### WAL 大小优化

```
PostgreSQL WAL 优化机制:
  - 记录操作 + 变更数据
  - 一次 UPDATE 改了 100B → WAL 只记录 100B + Header
  - 对于部分修改场景，WAL 大小远小于原始数据
```

### 关键代码位置

```c
// src/include/access/xlogrecord.h

// WAL 记录头
typedef struct XLogRecord {
    uint32    xl_tot_len;    // 总长度
    TransactionId xl_xid;    // 事务 ID
    XLogRecPtr xl_prev;      // 前一条记录位置
    uint8     xl_info;       // 记录类型
    RmgrId    xl_rmid;       // 资源管理器 ID
    pg_crc32c xl_crc;        // CRC 校验
} XLogRecord;

// 插入操作的 WAL 数据
typedef struct xl_heap_insert {
    OffsetNumber offnum;     // 行偏移
    uint8 flags;             // 标志位
    // 后面跟着实际的元组数据
} xl_heap_insert;
```

## 问题 3：WAL 中是否包含用户原始数据？

### 答案

**是的**，WAL 中**包含用户数据**，但只记录**变更的部分**。

### WAL 中用户数据的记录方式

```
INSERT 操作:
─────────────────────────────────────────────────────────────────────
INSERT INTO users (id, name, email) VALUES (1, 'Alice', 'a@b.com');

WAL 记录:
├── Header: ~24B
└── Data: id=1, name='Alice', email='a@b.com'  ← 用户数据
         大小: ~20B (只记录插入的行)

─────────────────────────────────────────────────────────────────────

UPDATE 操作:
─────────────────────────────────────────────────────────────────────
UPDATE users SET name = 'Bob' WHERE id = 1;
(只改了 name 字段，其他字段不变)

WAL 记录:
├── Header: ~24B
├── 旧值: name='Alice'  ← 用户数据 (用于回滚)
└── 新值: name='Bob'    ← 用户数据 (用于重放)
         大小: ~10B (只记录变更的字段)

─────────────────────────────────────────────────────────────────────

DELETE 操作:
─────────────────────────────────────────────────────────────────────
DELETE FROM users WHERE id = 1;

WAL 记录:
├── Header: ~24B
└── 被删除行数据: id=1, name='Bob', email='a@b.com'  ← 用户数据
         大小: ~20B
```

### 总结

| 问题 | 答案 |
|------|------|
| WAL 中有用户数据吗？ | **有** |
| 是完整原始数据吗？ | **不是**，是变更的数据 |
| INSERT 记录什么？ | 插入的行数据 |
| UPDATE 记录什么？ | 修改的字段（旧值 + 新值） |
| DELETE 记录什么？ | 被删除的行数据 |

## 问题 4：Shared Buffers 中保存的是什么？

### 答案

**Shared Buffers 保存的是数据页（Data Pages），即表数据的缓存。**

### Shared Buffers 结构

```
Shared Buffers (默认 128MB)
┌─────────────────────────────────────────────────────────────────────────┐
│ Buffer Array (缓冲区数组)                                                │
│ ├── Buffer 0: 8KB 数据页                                                 │
│ ├── Buffer 1: 8KB 数据页                                                 │
│ ├── Buffer 2: 8KB 数据页                                                 │
│ ├── ...                                                                  │
│ └── Buffer N: 8KB 数据页                                                 │
├─────────────────────────────────────────────────────────────────────────┤
│ Buffer Table (哈希表，快速查找)                                          │
│ Key: (表空间OID, 关系OID, 块号)                                          │
│ Value: Buffer ID                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 单个 Buffer 的内容

```
单个 Buffer (8KB)
┌─────────────────────────────────────────────────────────────────────────┐
│ Buffer Header (缓冲区头，元数据)                                          │
│ ├── buf_id: 缓冲区 ID                                                    │
│ ├── buf_state: 状态标志 (dirty, valid, pinned 等)                        │
│ ├── tag: (表空间OID, 关系OID, 块号)                                      │
│ ├── usagecount: 使用计数 (用于时钟扫描算法)                               │
│ └── wait_backend: 等待该 buffer 的后端进程                               │
├─────────────────────────────────────────────────────────────────────────┤
│ Page Data (页面数据，8KB)                                                 │
│ └── 实际的数据页内容                                                      │
└─────────────────────────────────────────────────────────────────────────┘
```

### Page Data 结构（单个数据页）

```
Page Data 结构 (8KB)
┌─────────────────────────────────────────────────────────────────────────┐
│ Page Header (24B)                                                        │
│ ├── pd_lsn: 最后修改该页的 WAL 位置                                      │
│ ├── pd_checksum: 页面校验和                                              │
│ ├── pd_lower: 空闲空间起始偏移                                           │
│ ├── pd_upper: 空闲空间结束偏移                                           │
│ ├── pd_special: 特殊空间起始偏移                                         │
│ └── pd_pagesize_version: 页面大小和版本                                  │
├─────────────────────────────────────────────────────────────────────────┤
│ Item Pointer Array (行指针数组)                                          │
│ ├── Item 0: (offset, length) 指向行数据                                  │
│ ├── Item 1: (offset, length) 指向行数据                                  │
│ └── ...                                                                  │
├─────────────────────────────────────────────────────────────────────────┤
│ Free Space (空闲空间)                                                     │
├─────────────────────────────────────────────────────────────────────────┤
│ Tuple Data (行数据，从下往上增长)                                          │
│ ├── Tuple Header (23B)                                                   │
│ │   ├── t_xmin: 插入该行的事务 ID                                        │
│ │   ├── t_xmax: 删除/更新该行的事务 ID                                   │
│ │   ├── t_cid: 命令 ID                                                   │
│ │   └── t_ctid: 当前行 TID (或新行 TID)                                  │
│ └── User Data: 实际用户数据                                              │
│     ├── id: 1                                                            │
│     └── name: 'Alice'                                                    │
├─────────────────────────────────────────────────────────────────────────┤
│ Special Space (特殊空间，如索引)                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### Shared Buffers 中存储的数据类型

| 类型 | 说明 |
|------|------|
| **表数据页** | 存储用户表的行数据 |
| **索引页** | 存储索引数据（B-tree, Hash 等） |
| **TOAST 页** | 存储超大的字段值 |
| **可见性映射页** | 跟踪行的可见性 |
| **空闲空间映射页** | 跟踪页内空闲空间 |

### 查找流程

```
用户查询: SELECT * FROM users WHERE id = 1

1. 计算目标页
   ┌─────────────────────────────────────────────────────────────────────┐
   │ 表空间 OID: 1663                                                     │
   │ 表 OID: 16384                                                        │
   │ 块号: 0 (根据 WHERE 条件和索引计算)                                   │
   └─────────────────────────────────────────────────────────────────────┘
            │
            ▼
2. 在 Buffer Table 中查找
   ┌─────────────────────────────────────────────────────────────────────┐
   │ Hash Key: (1663, 16384, 0)                                          │
   │ 查找 Buffer Table                                                   │
   └─────────────────────────────────────────────────────────────────────┘
            │
            ├── 命中 ──▶ 返回 Buffer ID ──▶ 读取页面数据
            │
            └── 未命中 ──▶ 从磁盘读取页面到空闲 Buffer
```

### 缓冲区状态

```
Buffer 状态标志:
┌─────────────────────────────────────────────────────────────────────────┐
│ BM_DIRTY        - 页面已修改，需要刷盘                                   │
│ BM_VALID        - 页面数据有效                                           │
│ BM_TAG_VALID    - buffer tag 有效                                        │
│ BM_IO_IN_PROGRESS - 正在进行 IO 操作                                     │
│ BM_PIN_COUNT_WAIT_BUF - 有进程在等待该 buffer                           │
│                                                                          │
│ PIN 计数: 有多少个后端进程正在使用该页面                                  │
│ - PIN > 0 时，该页面不能被置换                                           │
│ - PIN = 0 时，该页面可以被时钟扫描算法选中置换                           │
└─────────────────────────────────────────────────────────────────────────┘
```

### 关键代码位置

```c
// src/include/storage/buf_internals.h

// Buffer 描述符
typedef struct BufferDesc {
    BufferTag   tag;            // (表空间OID, 关系OID, 块号)

    int         refcount;       // 引用计数

} BufferTag;
```

### 总结

| 问题 | 答案 |
|------|------|
| Shared Buffers 保存什么？ | **数据页** (8KB 页面) |
| 数据页包含什么？ | Header + 行指针 + 行数据 |
| 大小？ | 默认 128MB |
| 单页大小？ | **8KB** |
| 查找方式？ | Hash Table (表空间OID, 表OID, 块号) |

## 问题 5：一张表在内存中的布局是怎样的？

### 答案

一张表在内存中由**多个 8KB 页面**组成，每个页面存储多行数据。

### 表的整体布局

```
表文件 (物理存储)
┌─────────────────────────────────────────────────────────────────────────┐
│ Block 0 (8KB)   Block 1 (8KB)   Block 2 (8KB)   ... Block N             │
└─────────────────────────────────────────────────────────────────────────┘
      │              │              │
      ▼              ▼              ▼
┌──────────┐   ┌──────────┐   ┌──────────┐
│ 页面 0   │   │ 页面 1   │   │ 页面 2   │   ...
│ 行 1-10  │   │ 行 11-20 │   │ 行 21-30 │
└──────────┘   └──────────┘   └──────────┘

加载到 Shared Buffers 后:
┌─────────────────────────────────────────────────────────────────────────┐
│ Buffer 0 (页面0)  Buffer 1 (页面1)  Buffer 2 (页面2)  ...                │
└─────────────────────────────────────────────────────────────────────────┘
```

### 单个页面的详细结构

```
单个页面 (8KB) 详细结构
┌─────────────────────────────────────────────────────────────────────────┐
│ 低地址                                                                  │
│ ┌─────────────────────────────────────────────────────────────────────┐│
│ │ Page Header (24B)                                                   ││
│ │ ├── pd_lsn (8B): 最后修改的 WAL 位置                                ││
│ │ ├── pd_checksum (2B): 页面校验和                                    ││
│ │ ├── pd_flags (2B): 标志位                                           ││
│ │ ├── pd_lower (2B): 空闲空间起始 (行指针数组结束位置)                 ││
│ │ ├── pd_upper (2B): 空闲空间结束 (行数据起始位置)                     ││
│ │ ├── pd_special (2B): 特殊空间起始                                   ││
│ │ └── pd_pagesize_version (2B): 页面大小和版本                        ││
│ └─────────────────────────────────────────────────────────────────────┘│
│ ┌─────────────────────────────────────────────────────────────────────┐│
│ │ Item Pointer Array (行指针数组)                                      ││
│ │ ├── ItemId 0 (4B): (offset=8160, len=32, flags=normal)              ││
│ │ ├── ItemId 1 (4B): (offset=8128, len=28, flags=normal)              ││
│ │ ├── ItemId 2 (4B): (offset=8100, len=28, flags=normal)              ││
│ │ └── ...                                                             ││
│ │                                                                     ││
│ │ 每个行指针 4B:                                                       ││
│ │ ├── lp_off (15b): 行数据在页面中的偏移                              ││
│ │ ├── lp_flags (2b): 状态 (normal/dead/unused)                        ││
│ │ └── lp_len (15b): 行数据长度                                        ││
│ └─────────────────────────────────────────────────────────────────────┘│
│ ┌─────────────────────────────────────────────────────────────────────┐│
│ │ Free Space (空闲空间)                                                ││
│ │        ↑ pd_lower 指向这里                                          ││
│ │        ... 空闲空间 ...                                              ││
│ │        ↓ pd_upper 指向这里                                          ││
│ └─────────────────────────────────────────────────────────────────────┘│
│ ┌─────────────────────────────────────────────────────────────────────┐│
│ │ Tuple Data (行数据，从页面底部向上增长)                               ││
│ │                                                                     ││
│ │ ┌───────────────────────────────────────────────────────────────┐  ││
│ │ │ Tuple N (最新插入)                                             │  ││
│ │ │ ├── HeapTupleHeaderData (23B)                                  │  ││
│ │ │ │   ├── t_xmin (4B): 插入事务 ID                               │  ││
│ │ │ │   ├── t_xmax (4B): 删除/更新事务 ID                          │  ││
│ │ │ │   ├── t_cid (4B): 命令 ID                                    │  ││
│ │ │ │   ├── t_ctid (6B): 当前/新行 TID                             │  ││
│ │ │ │   ├── t_infomask (2B): 标志位                                │  ││
│ │ │ │   └── t_bits: NULL 位图                                      │  ││
│ │ │ └── User Data: id, name, email...                              │  ││
│ │ └───────────────────────────────────────────────────────────────┘  ││
│ │ ┌───────────────────────────────────────────────────────────────┐  ││
│ │ │ Tuple 0 (最早插入)                                             │  ││
│ │ └───────────────────────────────────────────────────────────────┘  ││
│ └─────────────────────────────────────────────────────────────────────┘│
│ ┌─────────────────────────────────────────────────────────────────────┐│
│ │ Special Space (特殊空间，索引页使用)                                 ││
│ └─────────────────────────────────────────────────────────────────────┘│
│ 高地址                                                                  │
└─────────────────────────────────────────────────────────────────────────┘
```

### 行插入示例

```
创建表: CREATE TABLE users (id INT, name VARCHAR(20), email VARCHAR(30));

插入数据:
INSERT INTO users VALUES (1, 'Alice', 'alice@test.com');
INSERT INTO users VALUES (2, 'Bob', 'bob@test.com');
INSERT INTO users VALUES (3, 'Charlie', 'charlie@test.com');

页面布局:
┌─────────────────────────────────────────────────────────────────────────┐
│ Page Header (24B)                                                       │
│ pd_lower = 36  (24 + 3*4 = 36, 行指针数组结束位置)                       │
│ pd_upper = 8060 (8192 - 3*44 = 8060, 行数据起始位置)                     │
├─────────────────────────────────────────────────────────────────────────┤
│ Item Pointer Array                                                      │
│ ├── ItemId 0: offset=8148, len=44, flags=normal  → 指向 Tuple 0        │
│ ├── ItemId 1: offset=8104, len=44, flags=normal  → 指向 Tuple 1        │
│ └── ItemId 2: offset=8060, len=44, flags=normal  → 指向 Tuple 2        │
├─────────────────────────────────────────────────────────────────────────┤
│ Free Space (8060 - 36 = 8024B 空闲)                                      │
├─────────────────────────────────────────────────────────────────────────┤
│ Tuple 2: (3, 'Charlie', 'charlie@test.com')  ← 最后插入，在最下面       │
│ Tuple 1: (2, 'Bob', 'bob@test.com')                                      │
│ Tuple 0: (1, 'Alice', 'alice@test.com')   ← 最先插入，在最上面          │
└─────────────────────────────────────────────────────────────────────────┘

注意:
- 行指针从上往下增长
- 行数据从下往上增长
- 新行插入到行数据区域的底部
```

### UPDATE 操作的页面变化

```
初始状态:
┌────────────────────────────────────────────────────────────────────────┐
│ ItemId 0 → Tuple 0: (1, 'Alice', 'alice@test.com')                     │
│ ItemId 1 → Tuple 1: (2, 'Bob', 'bob@test.com')                         │
│ ItemId 2 → Tuple 2: (3, 'Charlie', 'charlie@test.com')                 │
└────────────────────────────────────────────────────────────────────────┘

执行: UPDATE users SET name = 'David' WHERE id = 2;

更新后:
┌────────────────────────────────────────────────────────────────────────┐
│ ItemId 0 → Tuple 0: (1, 'Alice', 'alice@test.com')                     │
│ ItemId 1 → Tuple 1: (2, 'Bob', 'bob@test.com')      ← 旧行，标记为旧   │
│                t_ctid = (0, 3)  ← 指向新行                             │
│ ItemId 2 → Tuple 2: (3, 'Charlie', 'charlie@test.com')                 │
│ ItemId 3 → Tuple 3: (2, 'David', 'bob@test.com')    ← 新行            │
└────────────────────────────────────────────────────────────────────────┘

MVCC 机制:
- 旧行保留，通过 t_ctid 指向新行
- 事务通过 t_xmin/t_xmax 判断行可见性
- VACUUM 时清理旧行
```

### 总结

| 问题 | 答案 |
|------|------|
| 表在内存中的基本单位？ | **8KB 页面** |
| 页面由什么组成？ | Header + 行指针数组 + 空闲空间 + 行数据 |
| 行指针作用？ | 定位行在页面内的位置 |
| 行数据增长方向？ | **从下往上** |
| 行指针增长方向？ | **从上往下** |
| 单行包含什么？ | Header (事务信息) + 用户数据 |
| UPDATE 如何处理？ | 旧行保留，插入新行，通过 t_ctid 关联 |
