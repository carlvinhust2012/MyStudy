# GPU Reduce 与模型训练中的梯度

## 1. GPU Reduce 运算

### 1.1 什么是 Reduce

Reduce 是对多份数据做**逐元素的归约运算**，把 N 个值合并成 1 个值。

```
4 张 GPU 各自算出了梯度:

GPU 0: [0.1, 0.2, 0.3]
GPU 1: [0.4, 0.5, 0.6]
GPU 2: [0.7, 0.8, 0.9]
GPU 3: [1.0, 1.1, 1.2]

Reduce (Sum) — 逐元素相加:
位置 0: 0.1+0.4+0.7+1.0 = 2.2
位置 1: 0.2+0.5+0.8+1.1 = 2.6
位置 2: 0.3+0.6+0.9+1.2 = 3.0
结果:   [2.2, 2.6, 3.0]
```

**逐元素**的意思：位置 0 和位置 0 算，位置 1 和位置 1 算，互不干扰。

### 1.2 Reduce 的运算类型

| 运算 | 公式 | 用途 |
|------|------|------|
| Sum | [a,b] + [c,d] = [a+c, b+d] | 最常用，梯度平均 |
| Max | max(a,c), max(b,d) | 求最大值 |
| Min | min(a,c), min(b,d) | 求最小值 |
| Prod | [a,b] * [c,d] = [a*c, b*d] | 求乘积 |

### 1.3 GPU 上怎么算

GPU 有成千上万个线程，天然适合做并行 reduce。经典做法是**树形归约**：

```
要对 [3, 1, 7, 0, 4, 1, 6, 3] 求 Sum:

Step 1: 相邻两个相加
  [3, 1, 7, 0, 4, 1, 6, 3]
    ↓  ↓    ↓  ↓    ↓  ↓    ↓  ↓
  [  4,     7,     5,     9  ]

Step 2: 再相加
  [4, 7, 5, 9]
    ↓  ↓    ↓  ↓
  [  11,        14      ]

Step 3: 最后相加
  [11, 14]
    ↓  ↓
  [  25  ]

结果: 3+1+7+0+4+1+6+3 = 25
```

8 个元素只需 3 步（log2(8)=3），1024 个元素只需 10 步。GPU 上成千个线程并行执行，速度极快。

### 1.4 Reduce 在 NCCL AllReduce 中的作用

```
分布式训练场景：每张 GPU 各自算出了本地 batch 的梯度，需要求平均

GPU 0: grad = [0.1, 0.2, 0.3, ...]
GPU 1: grad = [0.4, 0.5, 0.6, ...]
GPU 2: grad = [0.7, 0.8, 0.9, ...]
GPU 3: grad = [1.0, 1.1, 1.2, ...]

AllReduce(Sum):
  4 张卡通信后，每张卡都拿到所有梯度的和:
  [2.2, 2.6, ...]

然后除以 4 得到平均梯度，更新参数
```

## 2. 模型训练为什么需要计算梯度

### 2.1 梯度是什么

梯度就是**损失函数对每个参数的偏导数**，告诉每个参数应该往哪个方向调、调多少。

### 2.2 用一个直觉的例子

假设训练一个简单模型 `y = w * x`（预测房价 y，输入面积 x，参数 w 是单价）：

```
当前: w = 5.0

训练数据: x=100, 实际房价 y_true=800
模型预测: y_pred = 5.0 * 100 = 500

损失（误差）: loss = (500 - 800)² = 90000

问题: w 应该调大还是调小？
```

梯度告诉你答案：

```
梯度 = ∂loss/∂w = 2 * (y_pred - y_true) * x
                = 2 * (500 - 800) * 100
                = -60000

梯度为负 → 说明 w 太小了，应该增大
```

更新参数：

```
w_new = w - learning_rate * 梯度
      = 5.0 - 0.001 * (-60000)
      = 5.0 + 60
      = 65.0     ← 更接近真实单价

再次预测: y = 65 * 100 = 6500 ... 越来越近
```

### 2.3 为什么需要梯度而不是直接调

| | 不用梯度（随机调） | 用梯度 |
|--|-------------------|--------|
| 调参方向 | 瞎猜，可能越调越差 | 数学保证方向正确 |
| 收敛速度 | 极慢，甚至不收敛 | 稳定收敛到最优 |
| 大模型参数量 | 数百亿个参数完全无法调 | 每个参数都有明确的调整量 |

梯度的本质意义：在损失函数的等高线上，**梯度指向下降最快的方向**（梯度下降法）。

### 2.4 分布式训练中梯度的角色

大模型参数太多（几十亿），一张 GPU 放不下，需要多张 GPU 分片训练：

```
GPU 0: 模型前半部分 + 数据 batch 0~31
GPU 1: 模型后半部分 + 数据 batch 32~63

问题: 两张 GPU 各自只看了部分数据，算出的梯度是「有偏」的
  GPU 0 的梯度: 基于 batch 0~31 估算
  GPU 1 的梯度: 基于 batch 32~63 估算

解决方案: AllReduce 把两张卡的平均梯度同步
```

训练一步的完整流程：

```
1. 前向传播（Forward）
   GPU 0: 输入 batch 0~31 → 计算预测值 → 计算 loss
   GPU 1: 输入 batch 32~63 → 计算预测值 → 计算 loss

2. 反向传播（Backward）
   GPU 0: 从 loss 反推，算出每个参数的梯度 grad_0
   GPU 1: 从 loss 反推，算出每个参数的梯度 grad_1

3. 梯度同步（AllReduce）    ← NCCL 在这里工作
   GPU 0 和 GPU 1 交换梯度，各自得到 grad = (grad_0 + grad_1) / 2

4. 更新参数
   GPU 0: param -= lr * grad
   GPU 1: param -= lr * grad（和 GPU 0 更新的值一样）

5. 重复 1~4，loss 逐渐下降
```

## 3. 总结：Reduce 与梯度的关系

```
模型训练的核心循环:

  数据 → 前向传播 → 计算损失 → 反向传播 → 得到梯度 → 更新参数
                                                      ↑
                                              这里需要 Reduce
                                              （多 GPU 梯度同步）

关系链:
  多 GPU 各自算出本地梯度 → Reduce 求平均 → 所有 GPU 用相同梯度更新
       ↑                                              ↑
    反向传播                                          梯度下降
```

| 概念 | 作用 |
|------|------|
| 梯度 | 告诉每个参数该往哪个方向调、调多少，是优化模型的核心 |
| Reduce | 把多个 GPU 的梯度合并成一个，消除单卡偏差 |
| NCCL AllReduce | 高效地完成跨 GPU 的 reduce，并让所有卡拿到完整结果 |
