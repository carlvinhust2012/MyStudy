## 一、NVMe 架构概述

### 1.1 核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                      Host (主机)                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │  NVMe Driver │  │  Admin SQ   │  │   I/O SQ 0..n       │  │
│  │             │  │  (提交队列)  │  │  (I/O 提交队列)      │  │
│  │  - 队列管理  │  │             │  │                     │  │
│  │  - 中断处理  │  │  ┌───┐      │  │  ┌───┐ ┌───┐ ┌───┐  │  │
│  │  - 命令构建  │  │  │CMD│      │  │  │CMD│ │CMD│ │CMD│  │  │
│  │             │  │  └───┘      │  │  └───┘ └───┘ └───┘  │  │
│  └──────┬──────┘  └──────┬──────┘  └──────────┬──────────┘  │
│         │                │                     │             │
│         │  Doorbell Reg  │                     │  Doorbell   │
│         │  (Tail Doorbell)│                     │  (Tail DB)  │
│         ↓                ↓                     ↓             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              PCIe Interface (BAR)                    │   │
│  └─────────────────────────┬───────────────────────────┘   │
└────────────────────────────┼───────────────────────────────┘
                             │ PCIe TLP (Transaction Layer Packet)
                             ↓
┌─────────────────────────────────────────────────────────────┐
│                    NVMe Controller (SSD)                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │  Command    │  │  Admin CQ   │  │   I/O CQ 0..n       │  │
│  │  Processing │  │  (完成队列)  │  │  (I/O 完成队列)      │  │
│  │             │  │             │  │                     │  │
│  │  - 解析命令  │  │  ┌───┐      │  │  ┌───┐ ┌───┐ ┌───┐  │  │
│  │  - 访问闪存  │  │  │CPL│      │  │  │CPL│ │CPL│ │CPL│  │  │
│  │  - DMA 传输  │  │  └───┘      │  │  └───┘ └───┘ └───┘  │  │
│  └──────┬──────┘  └──────┬──────┘  └──────────┬──────────┘  │
│         │                │                     │             │
│         │  Completion      │                     │  Head Doorbell│
│         │  (更新 Head)     │                     │  (Host 写)   │
│         ↓                ↓                     ↓             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              NAND Flash Controller                   │   │
│  │         (闪存控制器 + DMA 引擎)                       │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 关键概念

| 概念 | 说明 |
|------|------|
| **SQ (Submission Queue)** | 提交队列，Host 写入命令，FIFO 结构 |
| **CQ (Completion Queue)** | 完成队列，Controller 写入完成状态 |
| **Doorbell** | 门铃寄存器，通知对方有新数据 |
| **PRP/SGL** | 物理区域页/散集列表，描述数据缓冲区 |
| **MSI/MSI-X** | 消息信号中断，Controller 通知 Host |

## 二、初始化时序

### 2.1 控制器初始化流程

```
Host                                           NVMe Controller
  │                                                    │
  │  1. PCIe 枚举 (Configuration Space)                │
  │ ────────────────────────────────────────────────>  │
  │     - 读取 BAR0 基地址                             │
  │     - 分配内存映射 I/O 空间                        │
  │     - 启用 Bus Mastering                           │
  │                                                    │
  │  2. 控制器复位 (CC.EN = 0)                         │
  │ ────────────────────────────────────────────────>  │
  │     写 Controller Configuration 寄存器             │
  │     使 Controller 进入复位状态                     │
  │                                                    │
  │  3. 等待控制器就绪 (CSTS.RDY = 0)                  │
  │ <────────────────────────────────────────────────  │
  │     轮询 Controller Status 寄存器                  │
  │     确认 CSTS.RDY = 0                              │
  │                                                    │
  │  4. 配置 Admin 队列                                │
  │ ────────────────────────────────────────────────>  │
  │     - AQA (Admin Queue Attributes)                 │
  │       ASQS: Admin SQ 大小 (条目数-1)               │
  │       ACQS: Admin CQ 大小 (条目数-1)               │
  │     - ASQ (Admin SQ Base Address)                  │
  │     - ACQ (Admin CQ Base Address)                  │
  │                                                    │
  │  5. 启用控制器 (CC.EN = 1)                         │
  │ ────────────────────────────────────────────────>  │
  │     设置 CC.EN = 1                                 │
  │     设置 CSS (Command Set Selected)                │
  │                                                    │
  │  6. 等待控制器就绪 (CSTS.RDY = 1)                  │
  │ <────────────────────────────────────────────────  │
  │     轮询 CSTS.RDY 直到 = 1                         │
  │     (最大等待 2秒，根据 Spec)                       │
  │                                                    │
  │  7. 发送 Identify 命令 (CNS = 01h)                 │
  │ ────────────────────────────────────────────────>  │
  │     获取 Controller 能力信息                       │
  │     (最大队列数、命名空间数量等)                    │
  │ <────────────────────────────────────────────────  │
  │     完成队列返回 Identify 数据结构                  │
  │                                                    │
  │  8. 发送 Set Features 命令                         │
  │ ────────────────────────────────────────────────>  │
  │     配置 Number of Queues                          │
  │     (申请 I/O SQ 和 CQ 数量)                        │
  │ <────────────────────────────────────────────────  │
  │     返回实际分配的队列数量                         │
  │                                                    │
  │  9. 创建 I/O 队列 (Create I/O SQ/CQ)               │
  │ ────────────────────────────────────────────────>  │
  │     Create I/O CQ (命令 05h)                       │
  │     - QID, Queue Size, Physical Address            │
  │     - Interrupt Vector (MSI-X entry)               │
  │ <────────────────────────────────────────────────  │
  │                                                    │
  │ ────────────────────────────────────────────────>  │
  │     Create I/O SQ (命令 01h)                       │
  │     - QID, Queue Size, CQID (绑定的 CQ)            │
  │     - Priority (Weighted Round Robin)              │
  │ <────────────────────────────────────────────────  │
  │     完成队列返回成功状态                            │
  │                                                    │
  │  10. 初始化完成，可开始 I/O 操作                    │
  │                                                    │
```

### 2.2 初始化时序详解

```
时间线 ──────────────────────────────────────────────────────────────>

PCIe 阶段:
├─ [T0] PCIe 链路训练完成 (LTSSM: L0)
├─ [T1] Host 读取 Configuration Space (Vendor ID, Device ID)
├─ [T2] Host 设置 Memory Base/Limit, 分配 BAR 空间
├─ [T3] Host 设置 Command Register (Bus Master Enable, Memory Space Enable)
└─ [T4] PCIe 枚举完成

控制器复位阶段:
├─ [T5] Host 写 CC.EN = 0 (开始复位)
├─ [T6] Controller 停止处理命令，清空队列
├─ [T7] Controller 设置 CSTS.RDY = 0 (复位完成)
└─ [T8] Host 轮询检测到 CSTS.RDY = 0

队列配置阶段:
├─ [T9]  Host 配置 AQA, ASQ, ACQ 寄存器
├─ [T10] Host 写 CC.EN = 1 (启用控制器)
├─ [T11] Controller 初始化 Admin 队列
├─ [T12] Controller 设置 CSTS.RDY = 1
└─ [T13] Host 检测到 CSTS.RDY = 1，开始发送 Admin 命令

能力协商阶段:
├─ [T14] Identify Controller (CNS=01h)
├─ [T15] Identify Namespace List (CNS=02h)
├─ [T16] Set Features (Number of Queues)
├─ [T17] Create I/O CQ 1..n
├─ [T18] Create I/O SQ 1..n
└─ [T19] 初始化完成，进入就绪状态
```

## 三、读操作完整时序

### 3.1 NVMe 读命令时序

```
Host Memory          Host Driver          PCIe          NVMe Controller          NAND
     │                    │                │                    │                 │
     │  1. 准备 PRP List    │                │                    │                 │
     │  (数据缓冲区描述)    │                │                    │                 │
     │◄─────────────────────│                │                    │                 │
     │                    │                │                    │                 │
     │  2. 构建 Read 命令   │                │                    │                 │
     │  CMD.SQID = 1        │                │                    │                 │
     │  CMD.CID = X         │                │                    │                 │
     │  CMD.OPC = 02h (Read)│                │                    │                 │
     │  CMD.NSID = Y        │                │                    │                 │
     │  CMD.SLBA = 起始 LBA │                │                    │                 │
     │  CMD.NLB = 块数      │                │                    │                 │
     │  CMD.PRP1 = 数据地址 │                │                    │                 │
     │◄─────────────────────│                │                    │                 │
     │                    │                │                    │                 │
     │  3. 写入提交队列     │                │                    │                 │
     │  SQ[Tail] = CMD      │                │                    │                 │
     │◄─────────────────────│                │                    │                 │
     │                    │                │                    │                 │
     │                    │  4. 写 Tail Doorbell                │                 │
     │                    │  (通知 Controller 有新命令)         │                 │
     │                    │────────────────────────────────────>│                 │
     │                    │                │                    │                 │
     │                    │                │  5. 读取 SQ 条目    │                 │
     │                    │                │  (DMA Read)         │                 │
     │◄─────────────────────────────────────────────────────────│                 │
     │                    │                │                    │                 │
     │                    │                │  6. 解析命令        │                 │
     │                    │                │  - 检查 LBA/长度    │                 │
     │                    │                │  - 映射到闪存页     │                 │
     │                    │                │                    │                 │
     │                    │                │  7. 从 NAND 读取数据│                 │
     │                    │                │─────────────────────────────────────>│
     │                    │                │                    │                 │
     │                    │                │                    │  8. 闪存读取完成  │
     │                    │                │◄─────────────────────────────────────│
     │                    │                │                    │                 │
     │                    │                │  9. DMA 写数据到 Host│                │
     │◄─────────────────────────────────────────────────────────│                 │
     │  数据写入 Host 内存  │                │                    │                 │
     │                    │                │                    │                 │
     │                    │                │  10. 写入完成队列   │                 │
     │                    │                │  CQ[Head] = CPL     │                 │
     │                    │                │  CPL.CID = X (匹配) │                 │
     │                    │                │  CPL.SC = 0 (成功)  │                 │
     │                    │                │                    │                 │
     │                    │                │  11. 生成中断 (MSI-X)│                │
     │                    │◄─────────────────────────────────────│                 │
     │                    │  ISR 执行       │                    │                 │
     │                    │                │                    │                 │
     │                    │  12. 读取 CQ 条目│                   │                 │
     │◄─────────────────────────────────────────────────────────│                 │
     │                    │                │                    │                 │
     │                    │  13. 写 Head Doorbell               │                 │
     │                    │  (通知 Controller 已处理完成)       │                 │
     │                    │────────────────────────────────────>│                 │
     │                    │                │                    │                 │
     │  14. 命令完成        │                │                    │                 │
     │  数据已在内存中      │                │                    │                 │
     │◄─────────────────────│                │                    │                 │
```

### 3.2 读操作时序详解

```
时钟周期 (PCIe 时钟域)
────────────────────────────────────────────────────────────────────────>

Host 侧:
    │
    │  命令准备
    ├─ [T0]  分配 PRP List，填充物理地址
    ├─ [T1]  构建 NVMe 命令结构体 (64 bytes)
    │        SQE: SQID=1, CID=42, OPC=02h, NSID=1, SLBA=0x1000, NLB=7
    ├─ [T2]  写入 SQ 槽位 (Host 内存操作)
    └─ [T3]  写 Tail Doorbell (PCIe MMIO Write)

PCIe 传输:
    │
    ├─ [T4]  Doorbell Write TLP 到达 Controller
    ├─ [T5]  Controller 发起 DMA Read (SQ 条目)
    ├─ [T6]  SQ 数据通过 PCIe 到达 Controller
    │
Controller 处理:
    │
    ├─ [T7]  命令解析，检查权限和有效性
    ├─ [T8]  闪存控制器调度读取操作
    ├─ [T9]  NAND 读取延迟 (典型 50-100μs)
    ├─ [T10] 闪存数据到达 Controller 缓冲区
    │
数据传输:
    │
    ├─ [T11] Controller 发起 DMA Write (数据到 Host)
    ├─ [T12] 数据通过 PCIe 写入 Host 内存
    ├─ [T13] DMA 完成，Controller 构建 CQE
    ├─ [T14] Controller 写 CQ 条目 (DMA Write)
    │
中断与完成:
    │
    ├─ [T15] Controller 发送 MSI-X Interrupt
    ├─ [T16] Host ISR 执行，读取 CQ 条目
    ├─ [T17] Host 处理完成状态，匹配 CID
    ├─ [T18] Host 写 Head Doorbell
    └─ [T19] 读操作完成，总耗时 ~100-150μs
```

## 四、写操作完整时序

### 4.1 NVMe 写命令时序（带 PLP/缓存）

```
Host Memory          Host Driver          PCIe          NVMe Controller        NAND/Cache
     │                    │                │                    │                 │
     │  1. 准备数据        │                │                    │                 │

     │◄─────────────────────│                │                    │                 │

     │◄─────────────────────│                │                    │                 │
     │                    │                │                    │                 │
     │  3. 构建 Write 命令  │                │                    │                 │
     │  CMD.OPC = 01h (Write)               │                    │                 │
     │  CMD.PRP1 = 数据地址 │                │                    │                 │
     │◄─────────────────────│                │                    │                 │
     │                    │                │                    │                 │
     │  4. 写入提交队列     │                │                    │                 │
     │                    │                │                    │                 │
     │                    │  5. 写 Tail Doorbell                │                 │
     │                    │────────────────────────────────────>│                 │
     │                    │                │                    │                 │
     │                    │                │  6. 读取 SQ 条目    │                 │
     │◄─────────────────────────────────────────────────────────│                 │
     │                    │                │                    │                 │
     │                    │                │  7. 解析命令，准备接收│                │
     │                    │                │                    │                 │
     │                    │                │  8. DMA 读取数据    │                 │
     │◄─────────────────────────────────────────────────────────│                 │
     │  数据被 Controller 读取              │                    │                 │

     │                    │                │  9. 数据到达，写入   │                 │
     │                    │                │  Controller Cache  │                 │
     │                    │                │─────────────────────────────────────>│

     │                    │                │                    │                 │
     │                    │                │  12. 写入完成队列   │                 │
     │                    │                │  CQ[Head] = CPL     │                 │
     │                    │                │                    │                 │
     │                    │                │  13. 生成中断       │                 │
     │                    │◄─────────────────────────────────────│                 │
     │                    │                │                    │                 │
     │                    │  14. 读取 CQ，写 Head Doorbell       │                 │
     │                    │────────────────────────────────────>│                 │
     │                    │                │                    │                 │
     │  15. 写完成 (但数据   │                │                    │                 │

     │◄─────────────────────│                │                    │                 │

注意: 对于非 FUA 写操作，第 11 步可能在完成通知之后异步执行
```

## 五、Admin 队列管理时序

```
创建 I/O CQ (Admin 命令 05h):
─────────────────────────────────────────

Host                                           NVMe Controller
  │                                                │
  │  Create I/O CQ 命令                            │
  │  ├─ DWORD10: QID (队列 ID)                      │
  │  ├─ DWORD10: QSIZE (队列大小)                   │
  │  ├─ DWORD11: PC (物理连续标志)                  │
  │  ├─ DWORD11: IEN (中断使能)                     │
  │  ├─ DWORD11: IV (中断向量)                      │
  │  └─ DPTR:    PRP1 (CQ 物理基地址)               │
  │ ────────────────────────────────────────────>  │
  │                                                │
  │  Controller 操作:                               │
  │  ├─ 验证 QID 和 QSIZE 有效性                    │
  │  ├─ 分配内部资源                                │
  │  ├─ 注册 CQ 物理地址                            │
  │  └─ 初始化 Head/Tail 指针                       │
  │                                                │
  │  完成状态 (成功/失败)                           │
  │ <────────────────────────────────────────────  │
  │
  等待 CQ 创建成功后，创建对应的 SQ

创建 I/O SQ (Admin 命令 01h):
─────────────────────────────────────────

Host                                           NVMe Controller
  │                                                │
  │  Create I/O SQ 命令                            │
  │  ├─ DWORD10: QID                               │
  │  ├─ DWORD10: QSIZE                             │
  │  ├─ DWORD11: PC                                │
  │  ├─ DWORD11: QPRIO (优先级)                     │
  │  ├─ DWORD11: CQID (绑定的 CQ ID)                │
  │  └─ DPTR:    PRP1 (SQ 物理基地址)               │
  │ ────────────────────────────────────────────>  │
  │                                                │
  │  Controller 操作:                               │
  │  ├─ 验证 CQID 已存在                            │
  │  ├─ 验证 QID 和 QSIZE 有效性                    │
  │  ├─ 建立 SQ→CQ 映射关系                         │
  │  └─ 初始化 SQ Tail Doorbell                     │
  │                                                │
  │  完成状态                                       │
  │ <────────────────────────────────────────────  │
```

### 5.2 删除队列时序

```
删除 I/O SQ (Admin 命令 00h):
```

## 六、多队列并行处理时序

### 6.1 多 SQ → 单 CQ 并行处理

```
场景: 4 个 I/O SQ 提交命令，共享 1 个 I/O CQ

时间 ─────────────────────────────────────────────────────────────────>

SQ 1 (高优先级):
├─ [T0]  CMD A ────────────────────────────┐
├─ [T20] CMD B ───────────────────────────┤
└─ [T40] CMD C ──────────────────────────┘

SQ 2 (中优先级):
├─ [T5]  CMD D ───────────────────────────┐
└─ [T25] CMD E ──────────────────────────┘

SQ 3 (中优先级):
└─ [T10] CMD F ──────────────────────────┐

SQ 4 (低优先级):
└─ [T15] CMD G ──────────────────────────┐

Controller 处理 (Weighted Round Robin):
├─ [T0]  收到 CMD A 通知，开始处理
├─ [T1]  收到 CMD D 通知，加入队列
├─ [T2]  收到 CMD F 通知，加入队列
├─ [T3]  收到 CMD G 通知，加入队列
├─ [T5]  CMD A 完成 (SSD 内部处理快)
├─ [T6]  开始处理 CMD B (SQ1 高优先级)
├─ [T10] CMD B 完成
├─ [T11] 开始处理 CMD D (轮询到 SQ2)
├─ [T15] CMD D 完成
├─ [T16] 开始处理 CMD F (轮询到 SQ3)
├─ [T20] CMD F 完成
├─ [T21] 开始处理 CMD G (轮询到 SQ4)
├─ [T25] CMD G 完成
├─ [T26] 开始处理 CMD C (回到 SQ1)
└─ [T30] CMD C 完成

CQ 完成顺序 (CID):
├─ [T5]  CPL (CID=A)
├─ [T10] CPL (CID=B)
├─ [T15] CPL (CID=D)
├─ [T20] CPL (CID=F)
├─ [T25] CPL (CID=G)
└─ [T30] CPL (CID=C)

注意: 完成顺序 ≠ 提交顺序，取决于闪存处理时间和优先级
```

### 6.2 多队列 Doorbell 合并优化

```
场景: Host 短时间内提交多个命令，优化 Doorbell 写入

Host 操作:
─────────────────────────────────────────

传统方式 (非优化):
├─ 写 SQ[0] = CMD1
├─ 写 Tail Doorbell (通知 Controller)  ←─ 立即触发
├─ 写 SQ[1] = CMD2
├─ 写 Tail Doorbell (通知 Controller)  ←─ 再次触发
├─ 写 SQ[2] = CMD3
└─ 写 Tail Doorbell (通知 Controller)  ←─ 第三次触发

优化方式 (批量提交):
├─ 写 SQ[0] = CMD1
├─ 写 SQ[1] = CMD2
├─ 写 SQ[2] = CMD3
├─ 写 SQ[3] = CMD4
└─ 写 Tail Doorbell (仅一次)           ←─ 通知最新 Tail

时序对比:
─────────────────────────────────────────

非优化:
    │ CMD1 │DB│ CMD2 │DB│ CMD3 │DB│
    └──────┘  └──────┘  └──────┘
    PCIe 事务多，延迟高

优化:
    │ CMD1 │ CMD2 │ CMD3 │ CMD4 │DB│
    └───────────────────────────────┘
    减少 PCIe 事务，降低延迟

Controller 响应:
─────────────────────────────────────────

非优化:
    ├─ [T0] 收到 DB，读取 CMD1
    ├─ [T2] 收到 DB，读取 CMD2
    └─ [T4] 收到 DB，读取 CMD3
    多次 DMA 读取，开销大

优化:
    ├─ [T0] 收到 DB，批量读取 CMD1-CMD4
    └─ [T2] 所有命令已读取
    单次 DMA 读取，效率高
```

## 七、中断处理时序

### 7.1 MSI-X 中断时序

```
NVMe Controller                              Host
       │                                        │
       │  1. 命令处理完成                        │
       │  2. 写入 CQ 条目                        │
       │                                        │
       │  3. 检查中断聚合配置                     │
       │     - 是否达到中断阈值?                  │
       │     - 是否超过中断延迟?                  │
       │                                        │
       │  4. 发送 MSI-X 中断                     │
       │  Memory Write TLP (特殊地址)            │
       │───────────────────────────────────────>│
       │                                        │
       │            5. PCIe Root Complex          │
       │               接收 TLP                   │
       │                                        │
       │            6. 触发 CPU 中断              │
       │               (特定向量号)               │
       │                                        │
       │            7. CPU 执行 ISR               │
       │                                        │
       │            8. ISR 读取 CQ 条目           │
       │            (处理所有待处理完成)          │
       │                                        │
       │  9. Host 写 CQ Head Doorbell            │
       │<───────────────────────────────────────│
       │     (通知 Controller 已处理完成)         │
       │                                        │
       │  10. Controller 可重新使用 CQ 槽位       │
       │                                        │

中断聚合 (Interrupt Coalescing):
─────────────────────────────────────────

场景: 高 IOPS 时减少中断风暴

配置参数:
├─ Aggregation Threshold: 8 (积累 8 个完成再发中断)
├─ Aggregation Time: 100μs (最大等待时间)
└─ Vector: MSI-X Table Entry 0

时序:
    │
    ├─ [T0]   完成 1 个命令 (不发中断，计数=1)
    ├─ [T5]   完成 2 个命令 (不发中断，计数=2)
    ├─ [T10]  完成 3 个命令 (不发中断，计数=3)
    ├─ ...
    ├─ [T35]  完成 8 个命令 (达到阈值，发中断)
    │         或
    ├─ [T100] 达到超时时间 (即使只有 5 个也发中断)
    │
    └─ [T101] Host 收到中断，批量处理 8 个完成

效果:
├─ 减少 CPU 中断次数 (8x 减少)
├─ 提高吞吐，增加延迟
└─ 适合高并发场景，不适合低延迟敏感场景
```

### 7.2 轮询 (Polling) 模式时序

```
场景: 低延迟要求的场景，避免中断开销

Host 操作:
─────────────────────────────────────────

方式 1: 忙等待 (Busy Polling)
    │
    ├─ 提交命令
    ├─ 写 Tail Doorbell
    │
    ├─ [循环开始]
    ├─ 读取 CQ Head 指针 (内存读取)
    ├─ 检查是否有新的 CQE
    ├─ [无新完成] ──→ 继续循环
    ├─ [有新完成] ──→ 退出循环
    │
    ├─ 处理 CQE
    └─ 写 Head Doorbell

时序:
    │
    ├─ [T0]  提交命令
    ├─ [T1]  开始轮询 (读内存)
    ├─ [T2]  读内存 (无完成)
    ├─ [T3]  读内存 (无完成)
    ├─ [T4]  读内存 (无完成)
    ├─ ...
    ├─ [T25] 读内存 (发现完成!) ← 约 25μs 后
    └─ [T26] 处理完成

CPU 开销: 高 (持续读取内存)
延迟: 极低 (无中断延迟)

方式 2: 混合模式 (Adaptive Polling)
─────────────────────────────────────────

    │
    ├─ 提交命令
    ├─ [T0-T10] 忙轮询 (期望快速完成)
    ├─ [T10]    未检测到完成
    ├─ [T10]    切换到中断模式
    ├─ 设置中断使能
    ├─ [等待中断]
    ├─ [T50]    收到 MSI-X 中断
    └─ [T51]    处理完成

优点:
├─ 短命令快速完成 (轮询)
├─ 长命令不浪费 CPU (中断)
└─ 自适应负载特征
```

## 附录: 关键数据结构

### NVMe 命令格式 (64 bytes)

```
Bytes   字段            说明
─────────────────────────────────────────
00      OPC             操作码 (01h=Write, 02h=Read)
01      FUSE            融合操作标志
02-03   CID             命令 ID (唯一标识)
04-07   NSID            命名空间 ID
08-15   Reserved
16-23   MPTR            元数据指针
24-31   PRP1            PRP 条目 1 (数据地址)
32-39   PRP2            PRP 条目 2 (列表地址或下一页)
40-47   CDW10           命令特定 (如 SLBA 低 32 位)
48-51   CDW11           命令特定 (如 SLBA 高 32 位)
52-55   CDW12           命令特定 (如 NLB, 读/写标志)
56-59   CDW13           命令特定
60-63   CDW14-15        命令特定
```

### NVMe 完成队列条目格式 (16 bytes)

```
Bytes   字段            说明
─────────────────────────────────────────
00-03   DW0             命令特定返回值
04-07   DW1             保留
08-11   DW2             状态字段 + SQ Head Pointer
        ├─ Bits 0-15:   SQHD (SQ Head Doorbell)
        └─ Bits 16-31:  Status (Phase Bit + SC + SCT)
12-15   DW3             命令 ID + SQ Identifier
        ├─ Bits 0-15:   CID (命令 ID，与提交匹配)
        └─ Bits 16-31:  SQID (来源 SQ)
```
